\doxysubsection{Description détaillée}
\mbox{\hyperlink{class_p_h_p_mailer_1_1_p_h_p_mailer_1_1_p_h_p_mailer}{PHPMailer}} POP-\/\+Before-\/\+SMTP Authentication Class. Specifically for \mbox{\hyperlink{class_p_h_p_mailer_1_1_p_h_p_mailer_1_1_p_h_p_mailer}{PHPMailer}} to use for RFC1939 POP-\/before-\/\+SMTP authentication. 1) This class does not support APOP authentication. 2) Opening and closing lots of \mbox{\hyperlink{class_p_h_p_mailer_1_1_p_h_p_mailer_1_1_p_o_p3}{POP3}} connections can be quite slow. If you need to send a batch of emails then just perform the authentication once at the start, and then loop through your mail sending script. Providing this process doesn\textquotesingle{}t take longer than the verification period lasts on your \mbox{\hyperlink{class_p_h_p_mailer_1_1_p_h_p_mailer_1_1_p_o_p3}{POP3}} server, you should be fine. 3) This is really ancient technology; you should only need to use it to talk to very old systems. 4) This \mbox{\hyperlink{class_p_h_p_mailer_1_1_p_h_p_mailer_1_1_p_o_p3}{POP3}} class is deliberately lightweight and incomplete, implementing just enough to do authentication. If you want a more complete class there are other \mbox{\hyperlink{class_p_h_p_mailer_1_1_p_h_p_mailer_1_1_p_o_p3}{POP3}} classes for PHP available.
